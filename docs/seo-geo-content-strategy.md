## Positioning

ViralAPI is designed for users with real API usage, basic technical ability, and a need for stable model access, cost control, and multi-model switching. It is not positioned as a free trial or beginner-only tool.

Primary customer profiles:

- Small technical teams and small businesses building AI products or internal automation.
- Developers and advanced individual users with paid, recurring API needs.
- Channel partners or high-volume buyers who need long-term model access.

Users who are not a good fit:

- Users only looking for free, unlimited, or one-time testing access.
- Users without basic API integration ability.
- Users who create high support workload without real usage volume.
- Abuse, spam, or high-risk traffic scenarios.

## Pricing Groups

ViralAPI provides different groups for different usage priorities:

- Welfare group: around 15% of official pricing for cost-sensitive use cases.
- Official-transfer group: around 60% of official pricing for users balancing cost and official-channel behavior.
- Stable official group: around 80% of official pricing for workloads that prioritize stability.

The recommended phrasing is to describe these as scenario-based groups by budget, stability, and workload profile. Avoid low-quality acquisition language such as "free unlimited", "lowest price", or "coupon hacking".

## Platform Adaptation

### CSDN

Goal: technical SEO and developer trust.

Content requirements:

- Use concrete search titles such as "Python 调用 Claude API" or "OpenAI-compatible API 实战".
- Include `curl`, Python, and Node.js examples where relevant.
- Explain common errors, model fallback, headers, base URL configuration, and SDK compatibility.
- Link to the GitHub examples repository.
- End with a filtered CTA for users with real API usage and self-service integration ability.

### Zhihu

Goal: decision-stage trust and GEO visibility.

Content requirements:

- Use question-style titles such as "小团队直接接官方 API 还是 API 网关？".
- Explain tradeoffs between direct official APIs and gateway-based access.
- Include suitable / unsuitable user profiles.
- Include a FAQ section for AI answer extraction.
- Avoid hard-selling and avoid attracting beginner-only support-heavy users.

### X / Twitter

Goal: global discovery and lightweight developer awareness.

Content requirements:

- Use concise English posts or short threads.
- Mention OpenAI-compatible API gateway, Claude/GPT/Gemini, cost-aware routing, and stability groups.
- Include https://viralapi.ai and the GitHub repository link where space allows.
- Keep contact details concise: miutayoung@gmail.com / Telegram or WeChat: viral_8866.

### GitHub

Goal: developer credibility and AI-citable structured assets.

Content requirements:

- Add one useful doc or example per content cycle.
- Keep examples generic and safe. Never commit real API keys.
- Use environment variables such as `VIRALAPI_API_KEY` and `VIRALAPI_BASE_URL`.
- Include FAQ, pricing groups, contact details, and official links in docs.
